The Problem

AI agents are powerful.
Managing them shouldn't be painful.

Running multiple Claude Code sessions across different tasks creates real friction that slows you down.

Missed Permission Requests

Claude Code is blocked in another tab waiting for approval, and you don't even know. Minutes wasted, momentum lost.

Invisible State

Which session is working? Which finished? Which errored out? You have no overview — just a row of identical terminal tabs.

Context Switch Tax

Cycling through tabs to check on each agent, remembering what each one is doing, rebuilding mental context every time.

Agent Team Complexity

A single session can spawn multiple teammates, creating a two-level hierarchy that's impossible to track manually.

Features

Everything you need to orchestrate
your AI workforce

Purpose-built for managing multiple Claude Code sessions with the speed and clarity you expect from a TUI.

Pod Management

All sessions live as Pods. Solo or Team mode, managed uniformly. Create, delete, and organize from one place.

Real-time Status

See working, idle, permission-needed, error, and done states at a glance. No more tab-cycling to check progress.

Permission Control

Approve or deny tool use requests directly from the dashboard. One keystroke to unblock your agents.

Chat Mode

Send messages to agents directly from Apiary. Diff-based response detection for reliable interaction.

Agent Teams

Auto-discover teammates within sessions. Two-level monitoring gives you visibility into the full hierarchy.

Adaptive Polling

Smart polling intervals based on agent state. Frequent checks when active, relaxed when idle. Efficient by default.

Git Worktree

Create pods with dedicated worktrees for isolated development. Each agent gets its own clean workspace.

Configurable

Customize behavior, polling intervals, keybindings, and display via a simple config.toml file.

How It Works

Up and running in minutes

Four steps from install to full orchestration of your AI coding agents.

1

Create a Pod

Spin up a new session with an optional worktree for isolated development.

apiary create issue-1234 --worktree ./worktrees/issue-1234

2

Monitor from the Dashboard

All your pods appear in a grid with real-time status updates. See who's working, who's done, and who needs attention.

3

Handle Permissions

When an agent needs tool approval, you'll see it immediately. Press a to approve or d to deny.

4

Chat with Agents

Need to redirect an agent or ask a question? Press c to enter chat mode and communicate directly.

Configuration

Make it yours

Customize Apiary with a simple TOML configuration file.

~/.config/apiary/config.toml
# Apiary Configuration

[general]
default_shell = "zsh"
log_level = "info"

[polling]
active_interval_ms = 1000
idle_interval_ms = 5000
permission_interval_ms = 500

[display]
show_elapsed_time = true
compact_mode = false
pods_per_row = 3

[worktree]
base_dir = "./worktrees"
auto_cleanup = true
